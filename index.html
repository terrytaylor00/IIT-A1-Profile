Project Idea

Overview

The chosen project will be a 3d game-like program in which users will be able to control a motor vehicle and drive it on different types of roads and scenarios. The main point of the program will be to properly teach Victorian driving rules and provide indication on whether or not a user has broken the law or otherwise how they have followed the road laws.

Motivation

The project will be useful as a learning tool for both new and ‘experienced’ drivers. Driving accidents and fatalities are one of the most dangerous aspects of modern Australian society, where despite lockdowns facing Victorians in 2021 “there were 237 road fatalities on Victorian roads in 2021 - 26 more than in 2020.”. With no foreseeable significant change on Victorian or Australian car-dependency in terms of infrastructure, culture or laws, it would be beneficial that for people who have to drive, that they atleast have access to appropriate tools to know how to be safe on the road.

Description

The idea for the potential program would likely be scaled to include differing aspects upon further development, however the full vision for the project can be detailed.

The 3d program would likely have scenarios in which users can choose from a starting UI menu. These scenarios will act like game levels, they will depict the most common dangerous scenarios that real life driving accidents may occur, for example how to navigate a large intersection. Upon choosing a scenario the user will most likely receive information about the relevant rules through UI elements, and will subsequently be given control of a 3d modelled car on a road emulating what a real driving environment may look like. How specifically these environments will look artistically will be touched upon further in this description. The player will then be prompted to navigate through the driving scenario controlling the car with the keyboard, if they cause any actions that would break laws or otherwise be dangerous the game will notify the user of their misaction through UI elements and give information on what to do next time.

Another aspect of the program would be a mode separate from choosing a scenario. Instead the game would feature an open driving mode where the user could spawn into an open driving environment with no objective. Instead the user would drive around and follow road rules, allowing them to experiment with what they can and can’t do, the system would still penalise or notify them of any mishaps.

In terms of the look of the game it is most likely that the program will be developed in a simplistic art style so features like roads, pedestrians and signs are easily identifiable and also allows development be emphasised on showing the most scenarios compared to art style. Using low polygon models and simplistic textures also means the program should be playable on many different devices, potentially even within a browser, allowing the highest level of accessibility to anyone.

Limitation of the project would include the exclusion of online connectivity or multiplayer gameplay. As the goal of the project is to promote positive driving behaviour, playing with other people would most likely have the opposite effect and encourage players to recklessly drive for fun. Another limitation would be the exclusion of different vehicles, unless specific to a discovered law or rule (like rules governing towing), the project would not feature different vehicles as this distracts from the driving safe aspect if people were able to use a faster car for example.

Tools and Technologies

The foundation of this game program would be a suitable game engine to develop the program in. Specifically it is expected that this program would be developed in Unity. Unity allows for multiplatform development which would be useful if wanting to reach the largest number of people on different platforms. Furthermore Unity being one of the most publicly utilised game engines means there are plenty of resources to be able to learn and develop the project online. It is expected that the team would use the 3d game aspect of unity and develop the game utilising either C# scripting language or the powerful visual scripting tools Unity has built in. As the program is expected to be in 3D, the creation of 3d assets will be needed. As such it is expected that Blender will be used. Blender is an open source 3d modelling tool that is also extremely popular. Blender will be used to create 3d models to be imported and used in Unity.

Skills Required

Skills required would include intermediary experience with Unity, having a solid foundational knowledge navigating the software and implementing basic gameplay elements. As such elements will need to be programmed a knowledge of C# or the visual scripting language would be necessary. As Unity is the most popular creator game engine, finding videos or resources to learn the software as well as its visual scripting aspect should be relatively easy. The knowledge of 3d modelling within blender would also be needed, specifically making models suitable and efficient for in-game rendering, similarly however as Blender is extremely popular finding online resources should not be difficult.

Knowledge that is specific to this project’s idea would be knowledge of Victorian driving laws. As these laws by nature should be of public interest and knowledge it should pose no issue in finding the relevant laws or rules in regard to driving. Websites such as vicRoads will most likely be the starting point in finding this knowledge.

Outcome

If the project is successful an overall increase in driver awareness and knowledge would be the most sought after outcome. Packaging essential knowledge in a captivating 3d experience would ensure that the program is further propagated within the general driving community. As such driving safety can only be achieved when everyone, at minimum, has a basic knowledge of the rules and importance of such rules.
